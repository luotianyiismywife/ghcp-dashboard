# 测试体系

## 概述

项目根目录下 `test/` 文件夹中有一系列独立的 Node.js 测试脚本，可在 VS Code 外部直接运行（`node test/*.js`）。这些测试不依赖 `vscode` 模块，直接读取文件系统来验证数据。

## 测试文件清单

| 文件 | 用途 | 能否独立运行 |
|------|------|-------------|
| `test/test_verify.js` | 语法/结构/功能的综合验证 | ✅ |
| `test/test_ai_stats.js` | AI Stats 数据验证 | ✅ |
| `test/test_aistats_diagnostics.js` | AI Stats 诊断分析 | ✅ |
| `test/test_fetcher_parity.js` | 数据获取器一致性测试 | ✅ |
| `test/test_heatmap.js` | 热力图数据测试 | ✅ |
| `test/test_late_may_probe.js` | 5 月数据探测 | ✅ |
| `test/test_may_28_31_raw_scan.js` | 5月28-31日原始扫描 | ✅ |
| `test/test_may_workspace_inspect.js` | 工作区检查 | ✅ |

## test_verify.js — 综合验证

验证以下内容：

1. **语法验证** — 检查所有 src/ 文件是否能被 Node.js 解析（`require()` 加载）
2. **结构验证** — 检查 `package.json` 的贡献点完整性
3. **函数签名验证** — 检查各模块导出的函数是否正确
4. **文件完整性** — 检查媒体文件是否存在

使用自制的 `test()` + `assert()` 框架，无第三方测试库。

```
输出格式:
  ✅ PASS: 测试名称
  ❌ FAIL: 测试名称 -- 错误信息
```

## test_ai_stats.js — AI Stats 数据验证

直接从工作区 `state.vscdb` 读取 AI Stats 数据并验证：

- 记录格式完整性
- 时间范围合理性
- 字符数非负性
- 跨工作区数据聚合
- WAL 文件读取

使用自制 `test()` 框架，结果统计 `passed/failed/skipped`。

## 测试特点

1. **无依赖**: 全部使用 Node.js 内置 `fs`、`path` 模块
2. **离线运行**: 不依赖 VS Code 或网络
3. **直接读 DB**: 复用 `accountDataFetcher.js` 中的相同策略（二进制扫描 + 正则）
4. **独立文件**: 每个测试文件可独立运行，互不依赖
